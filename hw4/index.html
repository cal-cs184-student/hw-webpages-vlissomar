Part 3: Handling collisions with other objects

  • Describe your implementation of handling collisions with spheres and planes.
 Sphere collisions are easily detected by getting the distance between the sphere’s origin and the point mass’s position. If it’s less than the radius of the sphere, there is an intersection, with the intersection tangent point at a well-defined radius away from the origin from origin to pm.position. Initialize the correction vector correction = tangent_point - pm.last_position, then add a scaled version of this to pm.last_position to get the current position.
 Plane collisions are a little trickier, but not that difficult with previous ray-plane intersections. First, by using dot products, it can be determined if a point is on the side of the plane with the normal or not. If the last_position and current position are on different sides, there is a collision. To account for the small displacement of the plane, I shift the tangent_point found through ray-plane intersection in the direction of the +/- normal vector, depending on which side of the plane that last_position was on. The remaining code is like sphere collision handling.

Part 4: Handling self-collisions

  • Describe your implementation of handling self-collisions.
 For hashing, I generate the coordinates in terms of the 3D box volumes as described in the specs, then multiply with large primes and XOR the results together. Building the hashmap is also rather simple; first, get the position hash for each PointMass, and if it’s in the map, append the pm to the end of the vector, and if not, create a new vector and add it to the map.
 The bulk of the work is in the self_collide function. Iterating over the vector retrieved from the map with the position hash, get the vector from the current PointMass to the other PointMass. If its norm is less than 2 * thickness, a collision has occurred, and a correction in the opposite direction of dist is accumulated. Outside the loop, apply the correction and scale by simulation_steps and the number of collisions.

  • Explain in your own words what a shader program is and how vertex and fragment shaders work together to create lighting and material effects.
 Vertex shaders organize information about the geometry of the scene, which is passed into fragment shaders later. They can also create physical distortions of the mesh, like displacement mapping. Fragment shaders compute the pixel at a specific point on the scene in terms of object/model space, using lighting calculations, model calculations from vertex shaders, and/or texture sampling to compute a color.
