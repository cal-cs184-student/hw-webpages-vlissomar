Overview

 In this homework, I implemented a global illumination algorithm as well as a BVH acceleration structure. Lighting calculations were made with the reflection equation provided in lecture, simplified with Monte Carlo estimators. My favorite part of this project is during global illumination, where recursive calls were able to render the indirect bounces of light that give life to the scene. I was also very impressed by how much an acceleration structure like a bvh could speed up the rendering process, so that normal shading time could be under a second. But even with this acceleration, global illumination can still take hours to render, especially at high sampling rates.

Part 1: Ray Generation and Scene Intersection

  • Walk through the ray generation and primitive intersection parts of the rendering pipeline.
 Ray Generation: Transform the endpoint of the ray from image coordinates to camera space, then world space. Using the camera position in world space, I can define a vector that corresponds to the ray with its origin and normalizing for direction. Then, set min and max t to the near and far clipping distances and return the ray. This ray generation method is called repeatedly on num_samples points within the pixel location given, and used to get a color sample. Since our pixel sampling is a uniform gridSampler, the MonteCarlo estimator essentially calculates an average of these sample colors to put in the sampleBuffer.
 Primitive Intersection: Primitive intersection is used to detect if there is an ‘earlier’ intersection of an object with the ray than is already recorded. If this is true, then the current primitive and other information about the intersection overwrites the information stored in the Intersection object, and r.max_t is updated. Detection here is primarily math.
  • Explain the triangle intersection algorithm you implemented in your own words.
 I use the Moller Trumbore algorithm, which conveniently finds the hit parameter t as well as barycentric coordinates (b1, b2). First, I check if t is within the valid range (indicating a plane intersection that is ‘early’ enough), then for valid barycentric coordinates (is the intersection within the triangle?). The latter checks b1, b2, as well as (1 - b1 - b2).

Part 2: Bounding Volume Hierarchy

  • Walk through your BVH construction algorithm. Explain the heuristic you chose for picking the splitting point.
&emspp;In my BVH construction algorithm, I decided to split along the longest axis of the bbox. Then, the Primitive* vector is sorted in place by the centroid coordinate of the splitting axis. By halving this vector, I get the primitives for my left and right child nodes, to be passed into recursive function calls.

Part 3: Direct Illumination

  • Walk through both implementations of the direct lighting function.
 Uniform Hemisphere sampling: I use a uniform Monte Carlo estimator of the form to calculate the direct lighting received by the surface. For each sample I incorporate, I use the uniform hemisphereSampler to get the solid angle direction in the object’s local coordinates. For all samples, the pdf = 1 / (2 * pi) since a hemisphere has 2 pi steradians. Each sample also has a cos𝜃= sample_dir.z, which can be verified with simple trigonometry. I transform this wi into world coordinates to make a ray for BVH intersection. If there is either no intersection or the emission magnitude is basically 0, then our radiance would add 0, so I early quit from the current loop iteration. Otherwise, I calculate the bsdf given the incoming & outgoing solid angle directions in local object coordinates, multiply by the emission at the intersection point, and multiply by cos𝜃, and accumulate in L_out. The function then returns L_out / pdf / num_samples to finish the estimator formula.  Light Importance sampling: The estimator here is similar to the previous one, except the pdf p(wi) is no longer constant, and must be calculated for each term.  Fortunately, a convenient light sampling function that can return the wi and pdf, as well as a distLight for ray intersection is provided. An important detail is that wi is given in worldspace, so to get cos𝜃, I use a dot product with the surface normal. Again, I do an intersection, this time also setting ray.max_t = distLight - EPS_F since the raycasting intersection here is not to see if the ray hits a light, but if there is any other object blocking hit_p from receiving direct light. Again, compute the term and accumulate in L_out. This time, 1/N is inside the term, as N is specific to the light type with point lights having N = 1.

Part 4: Global Illumination

  • Walk through your implementation of the indirect lighting function.
 First are base cases for r.depth == 0 or == 1, where the zero or one bounce radiance are simply returned. Then, sample the bsdf at the current intersection to get the local object space wi (the direction of the outgoing ray from the hit point) and pdf probability of the ray direction. Generate a ray at the next depth with this wi and hit_p, and use it to intersect the scene for the next bounce. If there is no intersection, just return L_out. Otherwise, run the Russian Roulette chance, where continuation probability = cpdf = 0.65 in my code. If the bounce is not terminated, do a recursive call to get the light at the second hit point. This is multiplied with other variables to calculate the light reflected at the first hit point and account for the probability of the sample, then returned in L_out.

Part 5: Adaptive Sampling

  • Explain adaptive sampling. Walk through your implementation of the adaptive sampling.
 Adaptive sampling can lower render time by not longer sampling when the pixel’s color seems to have converged. The logic behind it is from statistics, and a pixel is considered to have converged once the standard deviation or variance is below a certain tolerance threshold relative to the mean. In my implementation, I accumulate sums s1 and s2 as defined in the specs, then check for convergence every samplesPerBatch samples. If the pixel has converged, I break from the loop and divide f_n by the number of loop iterations, and update sampleCountBuffer. Then, update the pixel value.
