Overview

In this homework, I learned the basics of what's lying under the hood when I play 3D games. Each object is modeled with a 3D triangle mesh, glued together by halfedges. I was initially skeptical how halfedges could work since some rules seemed arbitrary. For example, how non HalfEdge meshelements only needed to point to any one of their related halfedges. But using these rules, I was able to traverse around the mesh to perform complex operations, like Loop Subdivision.

Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

  • Briefly explain de Casteljau’s algorithm and how you implemented it in order to evaluate Bezier curves.
Casteljau’s algorithm puts a slider between each consecutive pair of n control points to create n - 1 intermediary points, repeating until there is only 1 point left. These sliders use the same input t as a measure of how far along the intermediary point is between the preceding and following points. Then, to draw out the curve, we take the path traced by the last point as t slides from 0 to 1.
For my implementation, I am only responsible for evaluating each step (creating new control points). I iterate over the vector of 2D points while tracking the point from the previous iteration. Then, I simply plug in the previous point p0 and the current point p into our lerp expression: (1 - 5) * p0 + t * p, and append the new point into our returning vector.

    Part 2: Bezier surfaces with separable 1D de Casteljau

    • Briefly explain how de Casteljau algorithm extends to Bezier surfaces and how you implemented it in order to evaluate Bezier surfaces.
    To get a Bezier surface with de Casteljau’s algorithm, start with the typical algorithm along one dimension. Then, using a cross section of these curves as control points, apply the algorithm again along the remaining dimension. This produces a smooth curved surface, since each cross section should have points very close to the previous one, so each curve is connected to the previous one.
    The implementation in the homework only requires the code to find where the point is in 3D when given the interpolation ‘slider’ values, u and v. As outlined above, collect the control points from the 1D Bezier point at u from the inner vectors of controlPoints, then return the 1D Bezier point at v from these control points.

    Section II: Triangle Meshes and Half-Edge Data Structure

    Part 3: Area-weighted vertex normals

    • Briefly explain how you implemented the area-weighted vertex normals.
    My implementation assumes that the given vertex is not a boundary vertex. To compute area-weighted vertex normals, I iterate through the halfedges of the vertex using a while loop, updating the current ‘focus’ halfedge curr to be curr->twin()->next(). For each curr halfedge, I record the vertices of its associated triangle, then calculate the crossproduct of (B - A, C - A) where A, B, C are vertex positions. This cross product has a convenient quality of having a direction along the normal vector and a magnitude equal to twice the triangle area, and can be directly accumulated in our vertex’s normal vector. The constant scaling factor of 2 can be ignored since the function returns a normalized unit vector.
